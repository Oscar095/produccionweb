## PASO 1 — CONEXIÓN Y ANÁLISIS DE BASE DE DATOS (OBLIGATORIO ANTES DE CUALQUIER CÓDIGO)

**Conéctate a la base de datos SQL Server con las credenciales que te voy a proporcionar 
y ejecuta los siguientes pasos de exploración:**

### 1.1 Descubrir el esquema completo
```sql
-- Listar todas las tablas del esquema
SELECT TABLE_SCHEMA, TABLE_NAME, TABLE_TYPE
FROM INFORMATION_SCHEMA.TABLES
WHERE TABLE_TYPE = 'BASE TABLE'
ORDER BY TABLE_SCHEMA, TABLE_NAME;
```

### 1.2 Para cada tabla relevante, obtener columnas
```sql
SELECT TABLE_NAME, COLUMN_NAME, DATA_TYPE, IS_NULLABLE, COLUMN_DEFAULT
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_NAME IN (<tablas encontradas>)
ORDER BY TABLE_NAME, ORDINAL_POSITION;
```

### 1.3 Relaciones y claves foráneas
```sql
SELECT 
    FK.TABLE_NAME AS FK_Table,
    CU.COLUMN_NAME AS FK_Column,
    PK.TABLE_NAME AS PK_Table,
    PT.COLUMN_NAME AS PK_Column
FROM INFORMATION_SCHEMA.REFERENTIAL_CONSTRAINTS RC
JOIN INFORMATION_SCHEMA.TABLE_CONSTRAINTS FK ON FK.CONSTRAINT_NAME = RC.CONSTRAINT_NAME
JOIN INFORMATION_SCHEMA.TABLE_CONSTRAINTS PK ON PK.CONSTRAINT_NAME = RC.UNIQUE_CONSTRAINT_NAME
JOIN INFORMATION_SCHEMA.KEY_COLUMN_USAGE CU ON CU.CONSTRAINT_NAME = FK.CONSTRAINT_NAME
JOIN INFORMATION_SCHEMA.KEY_COLUMN_USAGE PT ON PT.CONSTRAINT_NAME = PK.CONSTRAINT_NAME;
```

### 1.4 Muestra de datos (top 5 por tabla)
```sql
SELECT TOP 5 * FROM <cada_tabla_relevante>;
```

**⚠️ IMPORTANTE**: No escribas UNA SOLA LÍNEA de código de la aplicación hasta haber 
completado este análisis. Preséntame un resumen del esquema encontrado y confirma 
conmigo las entidades antes de continuar.

## PASO 3 — MÓDULOS A DESARROLLAR (en este orden de prioridad)

### 🔴 MÓDULO 1: Dashboard de Producción
- KPIs: órdenes completadas, en proceso, pendientes, vencidas
- Eficiencia por centro de trabajo (actual vs capacidad)
- Alertas de paradas de máquina activas
- Gráficas de tendencia semanal/mensual

### 🔴 MÓDULO 2: Diagrama de Gantt de Capacidad
- Vista por centro de trabajo vs tiempo (días/semanas)
- Barras coloreadas por estado: En proceso / Pendiente / Parado / Completado
- Indicador visual de sobrecarga de capacidad (zona roja)
- Afectación de paradas de mantenimiento sobre la capacidad disponible
- Interactivo: hover para ver detalle de orden, click para editar
- Filtros: por centro de trabajo, rango de fechas, estado

### MÓDULO 3: Planeación y Priorización
- Vista tipo tablero por centro de trabajo
- Drag & drop para reordenar prioridad de órdenes
- Acciones por orden:
  - Marcar como cumplida
  - Suspender (con motivo)
  - Subir/bajar prioridad
  - Ver detalle completo
- Motor de cálculo: dado el tiempo estándar por orden y la capacidad disponible 
  del centro (descontando mantenimientos), calcular automáticamente qué órdenes 
  son alcanzables en la semana

### MÓDULO 4: Gestión de Mantenimiento
- Registro de tickets de mantenimiento
- Asociar parada a centro de trabajo + rango de fechas/horas
- Impacto automático sobre la disponibilidad en el Gantt
- Estados: Abierto / En progreso / Cerrado

### MÓDULO 5: Resumen Semanal para Supervisores
- Generación automática (puede ser manual o programada)
- Contenido del resumen:
  - Órdenes que DEBEN completarse esta semana (por prioridad)
  - Órdenes en riesgo (capacidad insuficiente)
  - Paradas de mantenimiento programadas
  - % cumplimiento de la semana anterior
- Formato exportable a PDF
- Opción de enviar por email (preparar endpoint, integración de correo opcional)

## RESTRICCIONES Y CONSIDERACIONES

1. **No romper los datos existentes**: todo acceso a tablas existentes debe ser 
   de solo lectura o con ALTER TABLE no destructivo. Las tablas nuevas van en 
   esquema separado `planeacion.*`
2. **AppSheet sigue funcionando**: la plataforma nueva convive con AppSheet 
   durante la transición — misma base de datos, sin conflictos
3. **Performance**: las consultas al Gantt pueden involucrar cientos de órdenes; 
   usa índices y paginación donde corresponda
4. **Responsive**: la interfaz debe funcionar en tablet (supervisores en planta)
